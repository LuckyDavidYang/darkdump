# AGENTS.md

## 目的

本仓库是一个小型 Python 命令行项目，主入口为 `darkdump.py`。
它通过 Ahmia 检索与查询词相关的 `.onion` 站点，并可在启用 Tor 代理后抓取页面内容，提取关键词、元数据、链接、邮箱、文档链接和图片。

在本仓库中工作的代理必须优先保证：

1. 改动小而可回退。
2. 文档与实际行为保持一致。
3. 先理解现有实现，再动手修改。

## 仓库结构

- `/darkdump/darkdump.py`
  主程序入口，包含参数解析、Ahmia 查询、Tor 检查、页面抓取和结果输出逻辑。
- `/darkdump/banner/banner.py`
  终端横幅输出。
- `/darkdump/headers/agents.py`
  请求时使用的静态 User-Agent 列表。
- `/darkdump/README.md`
  面向用户的安装、使用和 Tor 配置说明。
- `/darkdump/requirements.txt`
  运行依赖列表。

## 环境约束

1. 本项目默认 Python 版本为 `3.12`。
2. 所有新增文档、说明和命令示例都应以 `Python 3.12` 为准。
3. 若发现代码或依赖与 Python 3.12 不兼容，必须先指出风险，再进行修改。

## 工作方式

1. 修改前先阅读相关文件，避免只凭文件名猜测逻辑。
2. 优先做最小必要修改，避免顺手大改。
3. 这是一个依赖外部网络、Ahmia 页面结构和 Tor 状态的项目，测试应尽量避免真实联网。
4. 涉及 CLI 参数、抓取逻辑或输出行为时，必须同时检查 `README.md` 和本文件是否需要同步更新。

## 常用命令

- 安装依赖：`python3.12 -m pip install -r requirements.txt`
- 查看帮助：`python3.12 darkdump.py --help`
- 通过 Tor 抓取：`python3.12 darkdump.py -q "hacking" -a 10 --scrape --proxy`

## 本项目的修改检查清单

1. 修改 CLI 参数时，检查 `darkdump.py` 中的参数定义、错误提示和 `README.md` 中的示例是否一致。
2. 修改抓取逻辑时，检查 Ahmia 页面结构变化、缺字段页面、死链页面和超时页面的行为。
3. 修改图片处理时，检查无图片、相对路径图片、非法文件名和输出目录创建失败的情况。
4. 修改文本分析时，检查空文本、非英文文本、全是 HTML 噪音文本和极短文本的行为。
5. 修改 Tor 相关逻辑时，检查开启代理与关闭代理两条路径。
